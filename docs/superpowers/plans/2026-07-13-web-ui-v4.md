# margay ui v4 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Sidebar service tree, explicit dependency pairing with mock-mode support, and a full-height log pane — per `docs/superpowers/specs/2026-07-13-web-ui-v4-design.md`.

**Architecture:** Engine grows one conf flag (`uses_optional`), one `--use` value (`none`), and a `conf-json` introspection subcommand. `ui.py` consumes `conf-json` to expose per-service dependency metadata in `/api/state`, adds `/api/pair-options`, and forwards `service`/`use` through `/api/up`. `ui.html` regroups the sidebar into project → service → worktree, inserts a pairing step before ambiguous "up"s, and makes the log area flex-fill.

**Tech Stack:** bash (engine), Python stdlib http.server (UI server), vanilla JS single-file UI, bash test harness (`test/margay_test.sh`, `test/ui_test.sh`).

**Testing conventions:** `margay_test.sh` builds throwaway git repos + confs in a tmpdir with `MARGAY_HOME` overridden; `ui_test.sh` boots `ui.py` with a mock `MARGAY_BIN` and asserts on JSON via `curl + python3`. Follow existing `assert_eq`/`assert_contains` helpers. Run a suite with `bash test/margay_test.sh` (or `ui_test.sh`); expect final line `all passed`.

### Task 1: conf flag `service_<name>_uses_optional`

**Files:** Modify `lib/config.sh` (validate block, after the `uses_project` check ~line 46); Test `test/margay_test.sh`.

1. Failing test: conf with `service_web_uses_optional=2` → `margay up` (or direct `margay::config_load`) fails mentioning `uses_optional`; `=1` with `uses_project` set passes validation; `=1` **without** `uses_project`/`needs` fails ("requires a dependency").
2. Implement in `margay::config_validate` inside the per-service loop:
   ```bash
   val="$(margay::svc_var "$s" uses_optional)"
   if [[ -n "$val" ]]; then
     [[ "$val" == 0 || "$val" == 1 ]] \
       || { _cerr "service_${s}_uses_optional must be 0 or 1 (got '$val')"; return 1; }
     if [[ "$val" == 1 && -z "$(margay::svc_var "$s" needs)$(margay::svc_var "$s" uses_project)" ]]; then
       _cerr "service_${s}_uses_optional=1 requires service_${s}_needs or _uses_project"; return 1
     fi
   fi
   ```
3. Run suite → pass. Commit `feat(engine): validate service_<name>_uses_optional conf flag`.

### Task 2: `--use NAME=none` + optional empty-dep launch

**Files:** Modify `margay` (`margay::launch` ~lines 69-84, `--use` usage strings), `lib/engine.sh` (`margay::resolve_dep`); Test `test/margay_test.sh`.

Behavior (from spec): explicit `none` → dep env vars unset, allowed only when `uses_optional=1`; no candidate + optional → same empty path (main_port is **not** auto-fallback for optional services); no candidate + not optional → unchanged (main_port then error). Registry `uses` stays empty → null.

1. Failing tests (start hook `echo "DEP=[${API_URL:-unset}]"` into its log):
   - optional + `--use api=none` → service starts, log shows `DEP=[unset]`, CLI prints `api → none (optional)`.
   - non-optional + `--use api=none` → dies mentioning `uses_optional`.
   - optional, no live dep, no override → starts, `DEP=[unset]` (even with `main_port` set).
   - non-optional, no live dep, `main_port` set → unchanged: `DEP=[http://localhost:<main>]`.
2. Implement in `margay::launch` after computing `o`:
   ```bash
   optional="$(margay::svc_var "$svc" uses_optional)"
   if [[ "$o" == none ]]; then
     [[ "$optional" == 1 ]] || die "--use $depsvc=none needs service_${svc}_uses_optional=1"
     depsvc_skip=1
   fi
   ```
   Resolution: when `optional==1` call `resolve_dep` with empty main_port and tolerate failure (`out=""`); skip export/echo when `dep_url` empty; print `  $depsvc → none (optional)` instead. Guard the env export with `[[ -n "$dep_url" ]]`.
3. Suite green. Commit `feat(engine): --use NAME=none and optional dependencies (empty-dep launch)`.
